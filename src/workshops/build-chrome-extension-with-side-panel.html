Workshop: Build a Chrome Extension with a Side Panel

People and Code
At Your Disposal

Understand the data communication in a Chrome extension with a side panel.

Style: guided

Duration: 1h30min

Overview

Side panels are really useful to create a fully independent DOM for your Chrome extension. But they can be tricky to manage. In this workshop, we'll explore how you can reliably manage how to exchange data between the side panel and the content via the service worker.
  • Use the People and Code extension builder
  • Manage the state of the side panel
  • Understand the different contexts of an extension
  • Establish reliable communication between the contexts of the extension
  • Call an external API from an extension

Exercises

  1. Create an extension boilerplate

    • Clone and setup the People and Code Chrome extension generator
    • Create an extension with side panel named "My Dictionary" (name the directory in kebab-case)
    • Follow the instructions on screen and open the code in VS Code (see repo README for detailed instructions)
    • Explore the code, look out for the following:
      • The utilities setup in the root directory
      • The `support` directory with some extra tools to facilitate publication (not used here)
      • The extension code in the publish directory
      • The manifest (permission declarations, host permissions, the three different contexts: the background or service worker, the content scripts and the side panel)
      • The subdirectories in publish that store the code of the different contexts. Note that the side panel has its own CSS and script(s).
    • Load the extension locally on your Chromium browser (see repo README for detailed instructions), and run it as is. Open the different consoles and get a feel for who does what.

    Congratulations! You now have your own local extension.

  2. Explore communication between the contexts

  3. Call an external API and display the results

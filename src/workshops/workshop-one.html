Workshop one: HTML & CSS

Overview

To get started you need a development environment that, once set up, will let you experiment freely with HTML and CSS.

Therefore, let's:

  • Set up a work environment using VS Code
  • Clone a repo using Git and the CLI
  • Learn how to create a website from a template
  • Run the web site locally on your machine

Semantic HTML

We begin with a simple page and a few HTML elements to give it structure.

These are header, nav, main, article, aside and footer.

These elements were introduced to provide native support for ARIA landmark roles which help visitors using screen readers navigate web pages more easily. But these semantic elements benefit everyone. When developers and designers pay attention to good semantics - using HTML elements correctly - they can be confident their pages will be understood by visitors on different devices including screen readers.

The best way to learn semantic HTML is to write it.

  • Create a simple page using the elements referred to above
  • Check that they are valid using a screen reader
  • Consider the example of how to write HTML poorly using bad semantics

Bad semantics is common but it's not too hard to avoid. There are tools you can use to check your pages.

Benefits of writing semantic HTML

  • Accessible by more people on different devices
  • Code that is easier to understand
  • Likely results in lighter file sizes which increases performance and reduces emissions
  • Good for SEO (search engines give more weight to keywords inside headings, etc.)

AI

LLMs such as Claude and ChatGPT can check code. Alternatively, prompt them to generate the pages you need.

Prompt example

I would like you to create index.html and styles.css files. The HTML page should contain a header, footer and a main which contains an article and two asides (one for a table of contents - TOC - for the article, the other for links to related articles).

I would like the TOC to move above the article on smaller screens, and the related articles to move below. Please add basic layout styling e.g. padding, margins. Use the default fonts. Use variables for repeating css statements, e.g. --padding. Use rems for units. Please use a grid layout with grid areas.

Appendix

When developers talk about the Terminal, the command-line, BASH, shell, the CLI they are referring to one thing: the application that let's you give instructions to the operating system.

On a mac - whose operating system is macOS - this application is Terminal. When you open the Terminal, you are in the home directory (or folder, it's the same thing).

Command-line instructions

li
List the files in the current directory
cd ..
Go to the directory above
cd { name of a listed directory }
Go into the named directory
cd ~
Return to the home directory

How to launch (open) VS Code from the command line (Terminal)

In order to open the current project from the command line in VS Code on a mac :

  • Open VS Code
  • Type Command + Shift + P
  • Type shell command
  • Select Install 'code' command in PATH
  • Restart Terminal

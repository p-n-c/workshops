Workshop one: HTML & CSS

Overview

To begin, you need a development environment that, once set up, will let you experiment freely with HTML and CSS.

Therefore, let's:

  • Set up a work environment using VS Code
  • Clone a repo using Git and the CLI
  • Learn how to create a website from a template
  • Run the web site locally on your machine

Semantic HTML

We begin with a simple page and a few HTML elements to give it structure.

These are header, nav, main, article, aside and footer.

These elements were introduced to provide native support for ARIA landmark roles which help visitors using screen readers navigate web pages more easily. But these semantic elements benefit everyone. Good semantics - using HTML elements correctly - give developers and designers more control and greater confidence over how their pages will be perceived by visitors on different devices and at different screen sizes.

The best way to learn how to use semantic HTML is to write it.

  • Create a simple page using the elements referred to above
  • Check that they are valid using a screen reader
  • Consider the example below of how to write HTML poorly, that is with bad semantics

Bad semantics is common but it's not too hard to avoid.
